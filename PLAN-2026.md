# Live Shopping Search Insights Report — 2026 Edition

## Context

A fresh, standalone 2026 update of the Live Shopping Search Index study (prior edition: Dec
2025). Goal: re-run every angle from the original report on updated US data, add new and
surprising angles, and deliver a 10-15 page publication-quality PDF with rich annotated
visualizations.

Key decisions:
- Fresh full Google Ads Keyword Planner API pull. Carry over the prior search terms plus new
  2026 seeds from web research. Use only the last 48 months the API returns; do NOT reuse any
  historical search volumes. Index normalized to the first month of the fresh window = 100.
- US market only. All Korea content ignored.
- Cover all angles: overall LS index, live selling, Whatnot, HSN/QVC convergence, trust, live
  auctions; plus new angles: emerging platforms, vertical breakouts, event correlation.
- Output: Quarto -> PDF via Typst, styled with `_brand.yml` (red/gray palette).

## What was produced

Data window: June 2022 - May 2026 (48 months), US/English. ~7,700 seeds -> ~257k raw ideas ->
~79k unique keywords. 100% API success.

Headline finding: live-shopping search interest bottomed mid-2024, then rose ~49% in 2025;
peak in Jan 2025 coincided with the US TikTok ban drama, then held a higher plateau.

## Repository layout

- `pipeline-2026/` — data pipeline (Python).
  - `config.py` — shared constants (US/English, paths, output dir `output-2026/`).
  - `00_prep_seeds.py` — stack/clean/dedup seeds from `data/seeds_2026/`, batch for API.
  - `00b_test_api_pull.py` — validate API connection (news/pizza/weather).
  - `01_full_pull.py` — full keyword-ideas pull (run with the ads-api venv).
  - `02_clean.py` — validate, dedup, assert 48 months -> `df_clean.parquet`.
  - `03_stats.py` — 3-month rolling avg + per-keyword linear trend -> `rolling.parquet`,
    `kw_trend_stats.parquet`.
  - `04_build_report_tables.py` — build indices and tidy CSVs into `report-2026/data/`.
- `data/seeds_2026/` — seed CSVs (prior seeds + baskets + new 2026 seeds).
- `report-2026/`
  - `live-shopping-2026.qmd` — the report. `_brand.yml` — styling. `R/theme.R`,
    `R/make_charts.R` — chart code. `figures/` — generated PNGs. `data/` — tidy CSVs.
  - `research-notes.md` — sourced external facts (GMV, funding, events, market size).
  - `live-shopping-2026.pdf` — final deliverable (14 pages).

## Environments

- API pull scripts need `google-ads` + ads-api helpers -> run in the ads-api venv (Python 3.12):
  `cd ../ads-api && uv run python <abs path>/pipeline-2026/01_full_pull.py`.
- Clean/stats/build scripts run in this project's venv (Python 3.14):
  `uv run python pipeline-2026/02_clean.py` (etc.).
- Charts: `Rscript report-2026/R/make_charts.R`. Report: `quarto render report-2026/live-shopping-2026.qmd --to typst`.

## How to refresh next time

1. Add any new seeds to `data/seeds_2026/new_seeds_2026.csv`.
2. `00_prep_seeds.py` -> `01_full_pull.py` (ads-api venv) -> `02_clean.py` -> `03_stats.py` ->
   `04_build_report_tables.py`.
3. `make_charts.R`, then `quarto render`.
